This post is dedicated to the Chinese Remainder Theorem, specifically proving that it is valid. To do so, I should first explain a few definitions:

To start, you have the notation for congruency under modulo. This is seen in the form of

x ≡ y (mod n)

This means that under modulo n , the value x is equal to y. For instance, the following is true

5 ≡ 12 (mod 7)

since 12 - 7 = 5. In general, I find it easier to work directly with the mathematical representation , which can be done with the following statement

x ≡ y (mod n) ↔ ∃k∈ℤ. x = y + nk

This allows us to convert a congruency into a traditional math equation. This will be useful later on. Conceptually, it makes sense, as if x ≡ y (mod n) then x - y should be a multiple of n, and vice versa.

With those definitions, I can now introduce the version of CRM that I will prove. CRM states that if we assume the following two equations

x ≡ a (mod m)

x ≡ b (mod n)

Then if gcd(m, n) = 1, there will be a unique solution for x under (mod mn).

When I first saw this definition, it made sense to me on a high level, but I was unsure why gcd(m, n) = 1 was a needed requirement, and why the value for x would be true under (mod mn). However, there is a straightforward way to prove this by simply following the math and using the prior statements to convert congruences. In fact, the first thing we can do is convert the congruences into equalities to get

x = a + mk for some k∈ℤ

x = b + nd for some d∈ℤ

Next, we can substitute a + mk into the second equation to get

a + mk = b + nd

Rearranging the variables gives us

mk = (b - a) + nd

We can then convert back into a congruence to give us

mk ≡ (b - a) (mod n)

Before moving on, I must introduce another important idea, which is that the GCD of two numbers x and y can be expressed as a linear combination of the two, expressed as

ax + by = gcd(x, y) for some a∈ℤ, b∈ℤ

Now, we can look at our current equation

mk ≡ (b - a) (mod n)

The end goal is to solve for k, such that we can plug the value in to solve for x in the first equation. Therefore, we need to remove the m in front of k to reach our answer. Since we are working under (mod n), we could try to find a value c∈ℤ such that cm ≡ 1 (mod n), as that would leave us with only k. But how do we know such a value exists? Well, we can convert the congruence into an equality to get

cm = 1 + nz for some z∈ℤ

Which we can rewrite as

cm - nz = 1

This actually confirms that c must exist, as the above claims that 1 can be expressed as a linear combination of m and n, which is true if gcd(m, n) = 1. Looking back at the definition for CRM, we can now see why gcd(m, n) was provided as a condition. Since we now know c exists, we can multiply our congruence by it to get

cmk ≡ c(b - a) (mod n)

Which reduces to

k ≡ c(b - a) (mod n)

Huzzah! We now have a value we for k. Before we plug it into our original equation, we should first convert it back into an equality, giving us

k = c(b - a) + nh for some h∈ℤ

Plugging this into our original equation gives us

x = a + m[c(b - a) + nh]

Which simplifies to

x = a + mc(b - a) + (mn)h

Importantly, this is now in a form that can be converted back into a congruence under mn. We now have

x ≡ a + mc(b - a) (mod mn)

This confirms the fact that x is a unique solution under (mod mn) thus completing our proof (insert QED or □ or whatever it is we do these days).

Finally, as a sanity check, we can confirm both congruences are true for this value of x.

To do so, we look back at the original equality we found

x = a + m[c(b - a) + nh]

and notice that we can convert this into a congruence under (mod m), giving us

x ≡ a (mod m)

This matches the first congruence exactly, confirming it is valid! The second congruence is not as straightforward, but still pretty simple to confirm. We once again look back at the original equality we found

x = a + m[c(b - a) + nh]

and distribute m to get

x = a + mc(b - a) + n(mh)

Similarly to before, we can notice that this can be converted into a congruence, this time under (mod n). This gives us

x ≡ a + mc(b - a) (mod n)

Now remember that cm ≡ 1 (mod n), so we can replace mc with 1 in our congruence to get

x ≡ a + (b - a) (mod n)

which clearly simplifies to

x ≡ b (mod n)

This matches the second congruence exactly, meaning x ≡ a + mc(b - a) (mod mn) satsifies both congruences. With that, CRM is officially proven!

I particularly like this form of the CRM proof as it leads you there by following a simple chain of math. It essentially just unfolds itself as you convert between equalities and congruences, and doesn't really require much in the way of external logic.
